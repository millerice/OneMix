## 适用对象与目标

OneMix 适合以下场景：

- 电商上新：快速产出主图和详情图
- 老品翻新：在已有素材基础上迭代风格
- 批量运营：多槽位并行生成，统一下载归档

你可以完成：

- 上传商品资料并提取结构化关键信息
- 规划主图/详情图提示词并批量出图
- 单槽位重构提示词并单独再生图
- 复用已生成图片作为下一轮参考图

## 使用前准备

- 已安装 Python 3.10+、Node.js 18+、npm
- 网络可访问你使用的模型服务
- 已准备至少一张白底参考图（建议清晰、主体完整）
- 已准备模型 Key（DashScope 或火山引擎 ARK）

## 启动方式（本机）

### 1) 启动后端

```powershell
cd OneMix
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python backend\run_server.py
```

### 2) 启动前端

```powershell
cd frontend
npm install
npm run dev
```

浏览器打开终端提示地址（通常为 `http://127.0.0.1:5173`）。

## 页面总流程

建议按页面从上到下顺序操作：

1. 步骤一：提取商品信息
2. 步骤二：生成主图
3. 步骤三：生成详情图

页面底部有运行日志，可用于排查失败原因与查看执行状态。

## 步骤一：提取商品信息

### 1) 上传输入资料

- 可上传文本文件（`txt`/`md`/`csv`/`json`/`log` 等）
- 可上传商品图片用于 OCR
- 也可直接编辑“合并文本（可编辑）”

### 2) 执行提取

- 点击 `OCR识别（仅提取文本）`：将图片文字写入合并文本
- 点击 `提取关键信息`：生成结构化 JSON，并自动回填商品名/描述

### 3) 质量建议

- 若提取效果不理想，先手动修正文案再重新提取
- 商品卖点、规格参数、使用场景写得越完整，后续出图越稳定

## 步骤二：生成主图

### 1) 上传与管理参考图

- 支持多选上传白底图
- 每张图支持 `预览` / `删除` / `替换`
- 可填写“视角描述”（用于提示词规划）

### 2) 规划槽位提示词

- 点击 `规划主图槽位提示词`
- 在表格中检查并调整：
  - 参考图索引
  - 参考图缩略图（可点击放大）
  - 提示词文本

### 3) 生成策略

- `重构提示词`：仅重写当前槽位提示词
- `生图`：仅生成当前槽位
- `一键生成主图`：批量生成全部槽位

说明：单独 `生图` 只会替换当前槽位结果，不影响其他槽位。

### 4) 拖拽复用

- 可将“已生成主图”拖拽回步骤二上传区
- 松开后自动加入参考图列表，适合快速二次迭代

## 步骤三：生成详情图

步骤三与步骤二操作方式基本一致：

- 上传、预览、删除、替换参考图
- 规划详情图槽位并批量生成
- 按槽位单独重构与重生成

### 与步骤二联动规则

- 步骤二与步骤三共用同一组参考图数据
- 在任一步骤删除/替换图片，另一处会同步更新
- 可把步骤二主图结果拖拽到步骤三，作为详情图参考图

## Key 配置说明

页面顶部可保存两类 Key：

- DashScope Key
- 火山引擎 ARK（即梦）Key

建议至少配置一种与你当前策略匹配的 Key：

- 使用 `doubao_seedream_5` 时，必须配置 ARK Key
- 其他策略通常使用 DashScope Key

## 下载与结果管理

- 任务完成后，可点击 `下载 ZIP` 打包导出
- 建议按“日期-商品名-版本”命名归档
- 迭代过程中保留表现好的中间结果，便于复用参考图

## 常见问题（FAQ）

### Q1：点击生成后提示缺少 Key

请先在页面顶部保存对应服务 Key，再重新发起任务。

### Q2：任务中断或刷新后看不到进度

可重新发起任务恢复；已导出的结果通常仍可下载。

### Q3：拖拽图片没有反应

请确认从“生成结果卡片”拖拽，并在步骤二/三上传区域内松开。

### Q4：参考图索引对应关系不清楚

在槽位表“参考图”列查看缩略图，点击可放大确认。

## 最佳实践

- 先完善步骤一信息，再做提示词规划
- 先批量生成，再对不满意槽位单独精修
- 把优质结果回流为参考图，逐轮提升稳定性
- 每轮结束及时导出 ZIP，避免结果散落
